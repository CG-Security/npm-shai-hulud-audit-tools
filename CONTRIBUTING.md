# Contributing Guidelines

Thank you for your interest in contributing to this project.  
These guidelines ensure consistency and clarity for all contributors.

## How to Contribute

### 1. Reporting Issues
If you find a bug, unexpected output, or false detection:

- Open a GitHub Issue  
- Include reproduction steps  
- Do not include sensitive code or credentials  
- Provide the OS version and PowerShell version if relevant  

### 2. Submitting Enhancements
If you want to improve the scripts or documentation:

1. Fork the repository  
2. Create a feature branch (`feature/update-scan-logic`)  
3. Commit changes with clear messages  
4. Open a Pull Request  

### 3. Coding Standards

- Use PowerShell 5.1+ compatible syntax  
- Use `Write-Host` for clear output intended for users  
- Comment complex logic  
- Do not introduce external dependencies without discussion  

### 4. Documentation

If modifying scripts:

- Update `README.md` if behavior changes  
- Update relevant examples  
- Ensure new parameters or outputs are documented
